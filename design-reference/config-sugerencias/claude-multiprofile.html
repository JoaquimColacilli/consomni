Claude Code · Windows

Múltiples perfiles
con un solo CLI

Un ejecutable. Tres carpetas de config. Cero interferencia.

C:\Users\TuNombre\
.claude\claude │ settings.json │ skills\ │ projects\ │ .claude-max\claude-max │ settings.json │ skills\ │ projects\ │ .claude-team\claude-team settings.json skills\ projects\
La variable de entorno CLAUDE_CONFIG_DIR le dice al ejecutable claude.exe en qué carpeta guardar todo: settings, plugins, skills, memoria de proyectos e historial. Cambiando esa variable antes de ejecutar claude, cada alias opera con su propia configuración aislada — como si fueran instalaciones completamente separadas.

Setup — paso a paso

01

Crear las carpetas

Una carpeta por perfil. Si ya tenés ~/.claude configurado, podés copiar su settings.json como punto de partida. Si no, el CLI las inicializa solo la primera vez.

PowerShell
PS> mkdir "$HOME\.claude-max"
PS> mkdir "$HOME\.claude-team"
02

Abrir el PowerShell profile

El profile es un script que PowerShell ejecuta al arrancar. Ahí van las funciones. La variable $PROFILE ya sabe dónde vive — no hace falta buscar la ruta a mano.

PowerShell
# Crear el profile si no existe
PS> if (!(Test-Path $PROFILE)) {
New-Item -Force -Path $PROFILE
}

# Abrirlo en tu editor
PS> notepad $PROFILE
03

Agregar las funciones

Pegá esto al final del archivo. Cada función setea CLAUDE_CONFIG_DIR apuntando a su carpeta y luego invoca al CLI pasándole todos los argumentos con @args.

Microsoft.PowerShell_profile.ps1
function claude-max {
$env:CLAUDE_CONFIG_DIR = "$HOME\.claude-max"
claude @args
}

function claude-team {
$env:CLAUDE_CONFIG_DIR = "$HOME\.claude-team"
claude @args
}
04

Recargar el profile

El dot-source (.) ejecuta el profile en la sesión actual sin abrir una terminal nueva. Después de esto, claude-max y claude-team ya están disponibles.

PowerShell
PS> . $PROFILE
05

Verificar

Ejecutá cada alias y confirmá que arranca. Si querés ver qué carpeta está usando en ese momento, revisá la variable desde adentro del CLI o desde PowerShell.

PowerShell
PS> claude # usa ~/.claude
PS> claude-max # usa ~/.claude-max
PS> claude-team # usa ~/.claude-team

# Ver qué carpeta está activa ahora mismo
PS> $env:CLAUDE_CONFIG_DIR

Comportamiento a tener en cuenta

Después de llamar claude-max, la variable CLAUDE_CONFIG_DIR queda seteada en esa sesión de PowerShell. Si luego llamás claude directo (sin alias), va a seguir apuntando a .claude-max hasta que cierres la terminal o la limpies manualmente con Remove-Item Env:CLAUDE_CONFIG_DIR. En la práctica no es un problema si siempre usás los alias — pero conviene saberlo.
